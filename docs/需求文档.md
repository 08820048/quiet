

# 极简单页技术博客系统 - 需求文档

**文档版本**：1.0  
**日期**：2026-02-07  
**目标**：构建一个个人拥有的、极简风格的技术博客网站，采用纯前端 + 静态部署方式，所有内容通过 Markdown 文件维护。

### 1. 项目概述

**项目名称**：极简 Markdown 博客（暂定，可自定义）  
**核心理念**：模仿 Bear Blog 的极简美学（纯文字焦点、无 JS 干扰、页面极小），但实现单人使用、无后端、无数据库。  
**核心价值**：
- 更新文章只需修改/添加 Markdown 文件 → Git push → 自动部署
- 所有文章集中在一个页面，通过“见底自动加载”（无限滚动）展示
- 极致性能：页面大小小、加载快、无 trackers、无广告

**技术约束**：
- 纯静态站点（SSG）
- 部署平台：Vercel（免费 tier 足够）
- 内容源：文件夹内的 Markdown 文件
- 前端框架：Astro（强烈推荐）或其他轻量静态生成器

### 2. 功能需求

#### 2.1 核心功能（MVP）

| 优先级 | 功能点                          | 详细描述                                                                 | 实现方式预期                  |
|--------|---------------------------------|--------------------------------------------------------------------------|-------------------------------|
| P0     | Markdown 文章管理              | 所有文章存放在 `src/content/blog/*.md` 文件夹，支持 frontmatter（标题、日期、标签等） | Astro Content Collections    |
| P0     | 首页单页无限滚动文章列表       | 一个页面展示所有文章，按发布日期倒序；滚动到底部自动加载更多（分批渲染） | React + react-infinite-scroll-component（client:load） |
| P0     | 文章渲染                        | Markdown 内容转为 HTML，支持代码高亮、数学公式（KaTeX）、图片（相对路径或外链） | Astro Markdown/MDX + remark/rehype 插件 |
| P0     | 顶部导航栏                      | 固定/粘性导航：【首页】 【项目】 【友链】                             | Astro 布局组件 + 路由        |
| P1     | 项目页面                        | 静态页面，展示个人开源/侧项目列表（可使用 Markdown 编写）              | 独立 Markdown 文件或 Astro 组件 |
| P1     | 友链页面                        | 静态页面，列出友情链接（可 Markdown 格式编写，支持简单列表或卡片）     | 独立 Markdown 文件           |
| P1     | 文章元数据显示                  | 每篇文章显示：标题、发布日期、标签（可选阅读时长）                     | 从 frontmatter 读取          |
| P2     | RSS feed                        | 自动生成 `/feed.xml` 或 `/rss.xml`，包含所有文章摘要                   | Astro 内置 RSS 集成          |
| P2     | 暗黑模式支持                    | 系统/手动切换暗黑模式，保持极简（Tailwind dark: 前缀）                 | Tailwind CSS + localStorage  |
| P2     | 移动端响应式                    | 单栏布局，手机阅读舒适（字体大小、间距适配）                           | Tailwind 或纯 CSS            |

#### 2.2 非功能需求

| 类别       | 要求                                                                 |
|------------|----------------------------------------------------------------------|
| 性能       | 首页首屏加载 < 1s，Lighthouse 性能/SEO/可访问性 ≥ 95 分             |
| 页面大小   | 压缩后单个页面 < 50KB（目标模仿 Bear Blog ~3-10KB）                  |
| JS 使用    | 最小化，仅无限滚动组件需要 JS（Astro island 架构）                   |
| 可访问性   | 支持键盘导航、屏幕阅读器（语义化 HTML、alt 属性、对比度）            |
| SEO        | 自动生成 sitemap.xml、meta tags、canonical                           |
| 部署       | Vercel 自动部署，Git push 后几秒生效，支持自定义域名                 |
| 维护性     | Markdown 文件即内容，修改后无需重新构建整个站点（Vercel 增量构建）   |

### 3. 页面结构与路由

| 路由              | 描述                           | 内容来源                  | 备注                     |
|-------------------|--------------------------------|---------------------------|--------------------------|
| `/`               | 首页（文章无限滚动列表）       | 所有 blog/*.md            | 核心页面                 |
| `/projects`       | 项目页面                       | projects/*.md 或 hardcode | 可静态 Markdown          |
| `/links`          | 友链页面                       | links.md                  | 简单列表或 Markdown      |
| `/blog/[slug]`    | 单篇文章详情页（可选）         | blog/*.md                 | 可选：如果想加独立阅读页 |
| `/feed` 或 `/rss` | RSS/Atom feed                  | 自动生成                  | 可选但推荐               |

**注意**：首页采用单页无限滚动模式，所有文章内容直接内嵌渲染（不跳转到详情页），保持 Bear 式极简。如果文章非常多（>100篇），可考虑后期加“加载更多”按钮作为 fallback。

### 4. 数据模型（Markdown Frontmatter 示例）

```yaml
---

description: "一篇关于如何用 Astro 实现 Bear Blog 风格博客的记录"  # 用于 SEO 和 RSS
---
正文 Markdown 内容...
```

### 5. 推荐技术栈（首选方案）

- **框架**：Astro（最新版）
- **样式**：Tailwind CSS（或纯 CSS 保持更极简）
- **内容层**：Astro Content Collections（Markdown）
- **无限滚动**：React + react-infinite-scroll-component（Astro 支持 React 岛）
- **代码高亮**：Shiki 或 Prism（通过 rehype 插件）
- **数学公式**：KaTeX（remark-math + rehype-katex）
- **部署**：Vercel（GitHub 集成）
- **其他插件**（可选）：@astrojs/rss、@astrojs/sitemap、@astrojs/mdx

### 6. 开发 & 上线流程（简要）

1. `npm create astro@latest` → 选 Blog 模板 + Tailwind + React
2. 配置 Content Collections（src/content/blog/）
3. 实现 Layout.astro（带导航）
4. 首页 index.astro 中集成 React 无限滚动组件
5. 写项目/友链 Markdown
6. GitHub repo → Vercel 导入 → 部署
7. 自定义域名（可选）
8. 开始写文章：新增 .md 文件 → commit & push
