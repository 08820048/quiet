# [C++游戏开发基础]:基于new和delete的动态内存分配

讨论动态内存分配之前,先看看C++中前面两种种基本的内存分配类型:

- 静态内存分配用于静态和全局变量。这类变量的内存是在程序运行时一次性分配的,并持续整个程序的生命周期。
- 自动内存分配用于函数参数和局部变量,这类变量的内存是在进入相关作用域时执行分配的,并在退出作用域时释放,根据需求可以多次进行。
- 至于第三点,就是本文即将讨论的主题:动态内存的分配。

> 上面两种分配方式有两个共同点:
>
> 1. 变量/数组的大小必须在编译时已知  
> 2. 内存分配和释放是自动进行的  

 在大多数情况下,这样的逻辑可能没什么大问题。但是存在一些情况比较例外,比如,假设我们需要用一个字符串来保存某个人的名字,但是在输入结束之前,我们不知道它们的名字叫具体会有多长,同样的原理,比如我们从磁盘读取未知大小的数据记录时,这个记录在读取结束之前也是未知数。

在游戏开发中,比如游戏的中的怪物数量可能会随着时间变化,这样的数据对我们来说也是动态变化的,这就导致了 **编译时已知**这个条件很难满足。

你可能会想到使用提前预分配指定大小的容量来解决这些问题,也不是说不可行,但是存在空间浪费和内存溢出的风险。 

比如说,如果我们为每个名字分配 25 个字符的空间，但平均名字长度只有 12 个字符，我们就使用了实际所需的两倍以上的空间。

 大多数常规变量（包括固定数组）都是在称为堆栈的内存部分中分配的。程序可用的堆栈内存通常相当小,比如作为游戏开发中工具的`Visual Studio` 默认的堆栈大小为 1MB。如果你超过这个数字，就会发生堆栈溢出，操作系统可能会关闭程序。

> 对于许多程序来说,将内存限制在`1MB`的范围内是一个值得思考的问题,尤其是在图形开发领域。

## 单个变量的动态内存分配

要动态的分配单个变量,使用`new`运算符来实现:

```cpp
new int;
```

new 运算符使用该内存创建对象，然后返回一个包含已分配内存地址的指针。通常，我们会将返回值赋给我们的指针变量，以便以后访问分配的内存。

```cpp
int* ptr {new int};
```

然后使用解引用来访问内存,进行值的分配。

```cpp
*ptr = 9;
```

注意,由于堆分配对象需要先从指针获取对象的地址,然后再通过地址获取对象的值。这比访问栈分配的对象要慢一些,使用栈进行分配时,编译器知道栈分配对象的地址,可以直接访问该地址以获取值。

## 动态内存分配的基本工作原理

计算机拥有一定的内存供应用程序使用。当你运行一个应用程序时，操作系统会将该应用程序加载到部分内存中。

应用程序使用的这部分内存被划分为不同的区域,每个区域承担者不同的任务,其中一个区域存放的便是你的程序代码。另一个区域用来保证程序正常的运作,比如,记录函数调用的情况,创建和销魂全局和局部变量等等。

尽管如此,一般情况下还是会有一定的内存处于空闲状态,等待着程序的使用请求。 当你进行动态内存分配时,就是在请求操作系统这部分空闲的内存,希望可以从中请求一部分用来给你的程序使用.如果系统内存满足要求,那么它就会根据你的请求返回一个内存地址给你的程序。

从此刻开始,你的程序便拥有并可以随意使用这部分内存了,当程序不在需要时,还可以将这部分内存归还给操作系统以便随时待命,等候其他程序的使用请求。
与静态或者自动内存不同,**动态分配的内存是由程序本身负责申请和释放的**,也就是说,动态内存的管理权不再由操作系统控制,而是转交给了程序。

**核心小结**

> **栈上对象的分配和释放是自动进行的。**
>
> 我们不需要手动处理内存地址 —— 编译器生成的代码会自动完成这些工作。
>
> 
>
> **堆上对象的分配和释放则不是自动完成的。**
>
> 我们必须手动参与。也就是说，我们需要一种明确的方法来引用某个特定的堆对象，以便在适当的时候手动销毁它。
>
> 这种引用堆对象的方式，就是通过**内存地址**。
>
> 
>
> 当我们使用 new 操作符时，它会返回一个指针，指向新分配对象的内存地址。
>
> 我们通常会把这个返回值保存在一个指针变量中，这样之后就可以通过这个地址访问对象，并在需要时销毁它。

当你动态分配一个变量时,可以通过直接初始化或者统一初始化来初始化这个变量:

```cpp
int* ptr1 {new int(6)};
int* ptr2 {new int {6}};
```

当我们动态分配的变量使用结束后,需要明确的告诉C++释放这部分内存以便于内存重用。对于单个变量,可以使用`delete`关键字的非数组形式来完成释放。

```cpp
delete ptr;
ptr = nullptr;
```

注意:

- `delete`运算符实际上并没有删除任何东西,它只是将内存归还给操作系统。操作系统随后可以自由地将这些内存重新分配给另一个应用程序。
- 虽然语法上看起来像是在删除一个变量，但实际上并不是这样！指针变量仍然具有之前的作用域，并且可以像其他变量一样被重新赋值（例如赋值为 nullptr）。
- delete 只能用来释放通过 new（或 new[]）动态分配的内存。如果你对一个普通变量的指针或已经被释放的指针使用 delete，可能会导致程序崩溃、未定义行为或内存损坏等严重后果。

C++ 并**不保证**被释放的内存内容会发生什么变化，也**不保证**被删除的指针值会变成什么样。

在大多数情况下，**被归还给操作系统的内存中仍然保留着原来的数据**，而那个指针仍然**指向已经被释放的内存地址**。

指向已释放内存的指针被称为 **悬空指针（dangling pointer）**。对悬空指针进行解引用或再次删除，会导致**未定义行为**（undefined behavior）。

```cpp
#include <iostream>

int main()
{
    int* ptr{ new int }; // 动态分配一个整数的内存
    *ptr = 7; // 将一个值写入这块内存

    delete ptr; // 将这块内存归还给操作系统。此时 ptr 成为悬空指针（dangling pointer）

    std::cout << *ptr; // 解引用一个悬空指针会导致未定义行为（undefined behavior）
    delete ptr; // 尝试再次释放这块已释放的内存也会导致未定义行为

    return 0;
}
```

在上面的程序中，之前分配给内存的值 7 可能仍然存在，但也有可能该内存地址的值已经发生变化。

另外，也有可能这块内存已经被分配给了其他应用程序（或操作系统自身使用），如果你尝试访问这块内存，操作系统可能会因此终止程序。

如果多个指针指向同一块已释放的内存（即多个悬空指针），任何一个指针的错误访问都会导致 **未定义行为**。

```cpp
#include <iostream>

int main()
{
    int* ptr{ new int{} }; // 动态分配一个整数的内存
    int* otherPtr{ ptr }; // otherPtr 现在指向与 ptr 相同的内存地址

    delete ptr; // 将这块内存归还给操作系统。此时 ptr 和 otherPtr 都是悬空指针。
    ptr = nullptr; // ptr 现在是一个 nullptr

    // 然而，otherPtr 仍然是一个悬空指针！

    return 0;
}
```

针对上面的问题,下面是几个规避建议:

- 尽量避免多个指针指向同一块动态分配的内存。如果无法避免,需要明确哪个指针拥有这块内存(方便后续的释放处理),和哪些指针只能访问这块内存。
- 当删除一个指针时,如果该指针在删除后没有立即超出作用域,应该将该指针设置为`nullptr`。

## new操作符分配失败的情况

- 在向操作系统请求内存时，极少数情况下，操作系统可能没有足够的内存来满足请求。
- 默认情况下，如果 new 分配内存失败，会抛出一个 bad_alloc 异常。如果这个异常没有被正确处理,程序会因为未处理异常而直接崩溃。
- 许多情况下,抛出异常（或者让程序崩溃）并不是我们希望的结果，因此有一种 new 的替代形式，可以用来告诉 new 在无法分配内存时返回一个空指针。

这时候,可以通过在 new 关键字和分配类型之间添加常量 std::nothrow 来实现出现分配异常时返回一个空指针。

```cpp
int* val {new(std::nothrow)int};
```

在上面的例子中，如果 new 无法分配内存，它将返回一个空指针，而不是已分配内存的地址。

请注意，如果你随后尝试解引用这个指针，会导致未定义行为（最有可能的情况是，程序崩溃）。因此，最佳实践是在使用分配的内存之前进行空指针的判断，确保它们实际上成功了。

```cpp
int* val { new (std::nothrow) int{} }; // 请求分配一个整数大小的内存
if (!val) // 处理 new 返回空指针的情况
{
    // 在这里进行错误处理
    std::cerr << "无法分配内存\n";
}
```

## 空指针和动态内存分配

空指针（设置为 nullptr 的指针）在处理动态内存分配时特别有用。在动态内存分配的上下文中，空指针基本上表示“此指针没有分配到内存”。

这使得我们能够像这样有条件地分配内存：

```cpp
// 如果 ptr 尚未分配内存，进行分配
if (!ptr)
    ptr = new int;
```

- 删除空指针不会产生任何影响，因此没有必要先检查指针是否为空再进行 delete 操作。delete ptr; 会安全地忽略空指针，只有当指针指向有效内存时，才会执行内存释放。

```cpp
if (ptr) // 如果 ptr 不是空指针
    delete ptr; // 删除它
// 否则什么都不做
```

上面的写法是多余的,可以像下面这样写:

```cpp
delete ptr;
```

**最佳实践:**

> 对空指针使用delete是允许的,并且不需要对delete语句进行条件判断。

## 内存泄漏

- 当你使用 new 动态分配内存时，这些内存会一直保持分配状态，直到你显式地使用 delete 释放它。即使函数执行完毕，内存也不会自动释放，直到你手动释放它或者程序结束时，操作系统回收内存。
- 指针作为局部变量，它的作用域是有限的，即它会随着函数的执行结束而失效。但是，指针指向的动态内存（如果没有释放）会继续存在，直到你显式释放它。这种指针生命周期与动态内存生命周期的不匹配可能导致问题，比如内存泄漏或悬空指针。

考虑下面函数:

```cpp
void doSomething()
{
    int* ptr{ new int{} };
}
```

这个函数动态分配了一个整数的内存，但从未使用 delete 释放它。因为指针变量就像普通变量一样，当函数结束时，ptr 将超出作用域。

并且因为 ptr 是唯一持有动态分配整数地址的变量，当 ptr 被销毁时，动态分配的内存的地址就没有了。也就是说，程序现在“丢失”了指向动态分配内存的地址。

因此，这个动态分配的整数不能被删除。

这就叫做 **内存泄漏**。



内存泄漏发生在你的程序在没有释放内存的情况下失去了某个动态分配内存的地址。当发生这种情况时，程序无法删除动态分配的内存，因为它已经不知道内存在哪里了。

操作系统也无法使用这块内存，因为它被认为仍然在程序中使用。



内存泄漏在程序运行时会占用可用内存，导致此程序以及其他程序都可用的内存减少。具有严重内存泄漏问题的程序可能会消耗所有可用内存，导致整个机器运行缓慢甚至崩溃。

只有当程序终止时，操作系统才能清理并“回收”所有泄漏的内存。

只有当程序终止时，操作系统才能清理并“回收”所有泄漏的内存。

```cpp
int value = 5;
int* ptr{ new int{} }; // 分配内存
ptr = &value; // 旧地址丢失，导致内存泄漏
```

可以通过在重新赋值之前删除指针来修复此问题：

```cpp
int value{ 5 };
int* ptr{ new int{} }; // 分配内存
delete ptr; // 将内存返回给操作系统
ptr = &value; // 将指针重新赋值为 value 的地址
```

通过第二次使用 new 分配内存并将地址赋给同一个指针 ptr，会覆盖掉原先存储的地址，从而丢失对第一次分配内存的引用，导致内存泄漏。

```cpp
int* ptr{ new int{} };
ptr = new int{}; // 旧地址丢失，导致内存泄漏
```

在重新分配内存之前，确保先释放掉原来的内存（使用 delete），以防止丢失原始内存地址并引发泄漏。
